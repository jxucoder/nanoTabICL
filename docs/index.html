Stage 1

Understand each feature.

Each feature column is processed independently through a shared set-attention encoder. Labels are injected into training rows so every embedding becomes target-aware.

Column-by-column

Unlike most models that see all features at once, TabICLv2 treats each column as an independent set of values. A single shared encoder processes every column the same way.

Think of it like reading one column of a spreadsheet at a time — learning each feature's distribution before combining them later in Stage 2.

Shared ISAB

Target-aware embeddings

Each scalar value is linearly projected from 1 dimension to d = 64 dimensions. For training rows, a label encoding is added element-wise before any attention happens.

The label is one-hot encoded (e.g. [1, 0, 0, …] for class 0) and projected through its own learned linear layer to the same 64-dim space, then added to the value embedding.

Test rows skip this step entirely — they carry only the raw projected value. This asymmetry is what lets the model learn from labeled context later in Stage 3.

Set attention (ISAB)

The Induced Self-Attention Block compresses information in two steps: column values attend to a small set of learned inducing points, then the inducing points attend back to every value.

This is more efficient than full pairwise attention and captures the complete column distribution — mean, variance, shape — in a compact representation.

Stage 2

Features talk to each other.

Within every row, the column embeddings interact through transformer self-attention blocks so the model can capture cross-feature relationships.

Learnable CLS tokens attend to all features and compress each row into a fixed-size representation.

Stage 3

Learn from context.

A row-level transformer lets each test row attend exclusively to the labeled training rows — just like how an LLM attends to earlier tokens.

An MLP decoder converts the attended representation into final predictions. No gradient update happens — this is pure in-context learning.

Run it yourself.

The entire TabICLv2 implementation lives in a single Python file. Pretrain on synthetic tasks and evaluate in minutes.

Note: Any browser-based demo on this page uses a k-NN approximation, not the actual transformer. Run the Python code below for real TabICLv2 results. 

uv sync
uv run python -m nanotabicl.demo          # random-weight demo
uv run python -m nanotabicl.nano_model     # pretrain + eval
from nanotabicl import NanoTabICLClassifier, sample_scm_bnn_task

X, y = sample_scm_bnn_task(n_rows=640, n_features=6,
                           task="classification", seed=0)

clf = NanoTabICLClassifier(checkpoint="nano_tabicl.pt")
clf.fit(X[:512], y[:512])
pred = clf.predict(X[512:])
proba = clf.predict_proba(X[512:])
